# Контекст проекта planner

## Что это

**Семейный ежедневник** — веб-приложение для задач, мыслей и долгосрочных целей. Открывается с любого устройства по адресу https://mahmoudik-byte.github.io/planner/. Ставится на телефон как PWA.

Пользователь: **Eldar** (mahmoudik@gmail.com, GitHub mahmoudik-byte). Общение на русском, нейтральный тон без «ты»/«вы», без эмодзи.

## Модель данных (Supabase)

- `workspaces` — «команда» (семья). У одного юзера может быть несколько.
- `workspace_members(workspace_id, user_id, role)` — членство.
- `tasks` — задачи (`text`, `scheduled_at`, `repeat`, `priority`, `done`, `list_id`).
- `notes` — мысли (свободные заметки).
- `goals` — долгосрочные цели.
- `lists` — пользовательские списки (Покупки, Работа, Дом и т.д.) с `icon`, `color`.
- **RLS включена везде.** Доступ только к данным своих workspace через helper `my_workspaces()`.
- Атомарное создание workspace + членства — серверная функция `init_workspace(ws_name)`.

## Реализованные фичи

- Авторизация email + пароль (Supabase Auth, подтверждение email отключено).
- Вкладки: **Сейчас** (час вперёд) / **Сегодня** / **Завтра** / **Неделя** / **Мысли** / **Жизнь** / **Списки**.
- Добавление через FAB (`+`): задача, мысль или цель. У задачи — дата, время, повтор (daily/weekly/monthly), приоритет (обычный/важно/срочно), назначение списка.
- Чекбоксы, удаление, фильтрация по статусу.
- **Списки** — отдельная сущность с эмодзи и цветом. Чип-селектор в шапке, фильтрация всех вкладок по выбранному списку. Управление через вкладку «Списки» (карточки + модалка).
- PWA — иконка на главный экран, оффлайн-кэш оболочки (Supabase-запросы не кэшируются).

## Известные ограничения / что не сделано

1. **Приглашение в семейный workspace через UI** — нет. Сейчас добавлять других надо вручную через SQL Editor:
   ```sql
   insert into workspace_members (workspace_id, user_id, role)
   values ('WS_ID', 'USER_ID', 'member');
   ```
2. **Повторяющиеся задачи** — поле `repeat` есть, но автогенерация следующего вхождения после выполнения не реализована.
3. **Напоминания / пуш-уведомления** — нет. Идея: Telegram-бот.
4. **Кастомный домен** `planner.vintikshop.ru` — не настроен, пока github.io.
5. **Поиск, фильтр по автору, экспорт** — нет.
6. **Архивирование/переупорядочивание списков** — поле `archived` есть, UI нет. `position` есть, drag&drop нет.

## Принятые решения / стиль

- Без сборщиков и фреймворков — чтобы можно было редактировать любой файл и сразу видеть результат.
- Один глобальный `state` в app.js, перерисовка через `render()`.
- Все запросы к Supabase идут с фильтром по workspace_id; RLS — последняя линия защиты.
- Тёмная тема по умолчанию, светлой нет.
- Текст UI на русском.
